# نظام المصادقة والجلسات في ArabGram

## نظرة عامة

تم تحديث نظام المصادقة في ArabGram ليوفر:

1. **التحقق من البريد الإلكتروني الحقيقي** - أكواد مشفرة وآمنة
2. **حفظ الجلسات في قاعدة البيانات** - جلسات دائمة وآمنة
3. **تشفير قوي** - استخدام Argon2id للكلمات المرور والأكواس

## معمارية النظام

### 1. التسجيل (Signup)

```
المستخدم
   ↓
[صفحة التسجيل] → إدخال البيانات (الخطوات 1-4)
   ↓
[/api/auth/send-verification] → إنشاء حساب + إرسال رمز
   ↓
[صفحة التحقق] → إدخال الرمز (الخطوة 5)
   ↓
[/api/auth/verify-email] → التحقق من الرمز + تفعيل الحساب
   ↓
[صفحة تسجيل الدخول] → تسجيل الدخول بالبريد وكلمة المرور
```

### 2. تسجيل الدخول (Sign In)

```
[صفحة تسجيل الدخول]
   ↓
[/api/auth/[...nextauth]] → NextAuth Credentials Provider
   ↓
[قاعدة البيانات] → البحث عن المستخدم + التحقق من كلمة المرور
   ↓
[إنشاء جلسة] → حفظ الجلسة في قاعدة البيانات
   ↓
[الصفحة الرئيسية] → الوصول الآمن
```

### 3. حفظ الجلسة

```
جلسة المستخدم
   ↓
[NextAuth Database Strategy]
   ↓
[جدول Session في قاعدة البيانات]
   ↓
[التحقق من الجلسة عند كل طلب]
   ↓
[تحديث تلقائي للجلسة]
```

## قاعدة البيانات

### جدول Users

| الحقل | النوع | الوصف |
|------|------|-------|
| id | String | معرّف فريد |
| username | String | اسم المستخدم (فريد) |
| email | String | البريد الإلكتروني (فريد) |
| hashedPassword | String | كلمة المرور المشفرة (Argon2) |
| name | String | الاسم الكامل |
| emailVerified | DateTime | تاريخ التحقق من البريد |
| verificationCode | String | رمز التحقق المشفر |
| verificationCodeExpires | DateTime | انتهاء صلاحية الرمز |
| role | Role | دور المستخدم (USER/ADMIN) |
| image | String | صورة الملف الشخصي |
| bio | String | السيرة الذاتية |
| createdAt | DateTime | تاريخ الإنشاء |
| updatedAt | DateTime | تاريخ آخر تحديث |

### جدول Session

| الحقل | النوع | الوصف |
|------|------|-------|
| id | String | معرّف فريد |
| sessionToken | String | رمز الجلسة (فريد) |
| userId | String | معرّف المستخدم |
| expires | DateTime | انتهاء صلاحية الجلسة |

## نقاط الاتصال (API Endpoints)

### 1. إرسال رمز التحقق
```
POST /api/auth/send-verification

Body:
{
  "email": "user@example.com",
  "name": "أحمد",
  "username": "ahmed123",
  "password": "SecurePassword123"
}

Response (200):
{
  "message": "تم إرسال رمز التحقق بنجاح",
  "maskedEmail": "ah***@example.com"
}

Errors:
- 400: بيانات غير صحيحة
- 409: البريد أو اسم المستخدم مستخدم بالفعل
- 500: خطأ في الخادم أو البريد
```

### 2. التحقق من البريد الإلكتروني
```
POST /api/auth/verify-email

Body:
{
  "email": "user@example.com",
  "code": "123456"
}

Response (200):
{
  "message": "تم التحقق من البريد الإلكتروني بنجاح"
}

Errors:
- 400: رمز غير صحيح أو انتهت الصلاحية
- 404: المستخدم غير موجود
- 500: خطأ في الخادم
```

### 3. إعادة إرسال الرمز
```
POST /api/auth/resend-verification

Body:
{
  "email": "user@example.com"
}

Response (200):
{
  "message": "تم إعادة إرسال رمز التحقق بنجاح"
}

Errors:
- 400: البريد مفعل بالفعل
- 404: المستخدم غير موجود
- 500: خطأ في الخادم
```

### 4. تسجيل الدخول
```
POST /api/auth/callback/credentials

Body:
{
  "email": "user@example.com",
  "password": "SecurePassword123"
}

Response (200):
- تحويل إلى الصفحة الرئيسية
- حفظ الجلسة في قاعدة البيانات

Errors:
- 401: بريد أو كلمة مرور غير صحيحة
- 403: البريد لم يتم التحقق منه
```

## ميزات الأمان

### 1. تشفير كلمات المرور
- **الخوارزمية**: Argon2id (الأقوى حالياً)
- **تكلفة الذاكرة**: 64MB
- **عدد التكرارات**: 3
- **التوازي**: 1

```typescript
const hashedPassword = await hashPassword(password)
const isValid = await comparePassword(password, hashedPassword)
```

### 2. تشفير أكواس التحقق
- **الخوارزمية**: Argon2id (نفس كلمات المرور)
- **الطول**: 6 أرقام
- **الصلاحية**: 10 دقائق فقط
- **التخزين**: مشفر في قاعدة البيانات

### 3. حماية الجلسات
- **الاستراتيجية**: Database (قاعدة البيانات)
- **مدة الجلسة**: 30 يوماً
- **التحديث**: تلقائي عند كل طلب
- **الحذف**: عند تسجيل الخروج

### 4. التحقق من البريد الإلكتروني
- **الإلزامية**: مطلوبة قبل تسجيل الدخول
- **الرسالة**: تحتوي على رمز فريد
- **الرابط**: آمن ولا يحتوي على بيانات حساسة

## الاختبار

### اختبار التسجيل
1. اذهب إلى `/auth/signup`
2. أدخل البيانات الصحيحة
3. تحقق من البريد الإلكتروني
4. أدخل الرمز المُرسل
5. سيتم توجيهك إلى صفحة تسجيل الدخول

### اختبار تسجيل الدخول
1. اذهب إلى `/auth/signin`
2. أدخل البريد وكلمة المرور
3. سيتم حفظ الجلسة في قاعدة البيانات
4. ستتمكن من الوصول إلى الصفحات المحمية

### التحقق من الجلسة
```typescript
import { getServerSession } from "next-auth"
import { authOptions } from "@/lib/auth"

const session = await getServerSession(authOptions)
console.log(session.user) // معلومات المستخدم
```

## استكشاف الأخطاء

### المشكلة: "فشل إرسال البريد الإلكتروني"

**الأسباب المحتملة**:
1. متغيرات البريد غير صحيحة
2. كلمة مرور التطبيق (Gmail) غير صحيحة
3. عدم تفعيل المصادقة ذات العاملين (Gmail)
4. خادم SMTP غير متاح

**الحل**:
```bash
# تحقق من متغيرات البيئة
cat .env.local | grep EMAIL

# اختبر الاتصال بخادم SMTP
telnet smtp.gmail.com 587
```

### المشكلة: "الرمز غير صحيح"

**الأسباب المحتملة**:
1. إدخال رمز خاطئ
2. انتهاء صلاحية الرمز (10 دقائق)
3. استخدام رمز قديم

**الحل**:
- اطلب رمزاً جديداً
- تأكد من نسخ الرمز بشكل صحيح

### المشكلة: "البريد لم يتم التحقق منه"

**الأسباب المحتملة**:
1. لم تكمل عملية التحقق
2. الرمز انتهت صلاحيته

**الحل**:
- أكمل عملية التحقق من البريد
- اطلب رمزاً جديداً إذا لزم الأمر
